# Detecting & Treating Outliers

> "An outlier is an observation which deviates so much from the other observations as to arouse suspicions that it was generated by a different mechanism." — Douglas Hawkins

## What You Will Learn

- Identify outliers using Z-Score and Interquartile Range (IQR)
- Use robust machine learning methods like Isolation Forest
- Decide whether to drop, cap (winsorize), or transform anomalous data

## Step 1: The IQR Method

The IQR is the difference between the 75th percentile ($Q_3$) and the 25th percentile ($Q_1$). It is non-parametric, meaning it does not assume your data is normally distributed. 

An outlier is typically defined as a point outside:
- $Q_1 - 1.5 \\times IQR$
- $Q_3 + 1.5 \\times IQR$

```python
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns

# Seed data
np.random.seed(42)
data = pd.Series(np.random.normal(50, 10, 1000))
data = pd.concat([data, pd.Series([120, 135, -20])], ignore_index=True)

# Calculate IQR
Q1 = data.quantile(0.25)
Q3 = data.quantile(0.75)
IQR = Q3 - Q1

lower_bound = Q1 - 1.5 * IQR
upper_bound = Q3 + 1.5 * IQR

# Identify Outliers
outliers = data[(data < lower_bound) | (data > upper_bound)]
print(f"Detected {len(outliers)} outliers using IQR.")

# Visualization
plt.figure(figsize=(10, 4))
sns.boxplot(x=data, color='lightskyblue')
plt.axvline(lower_bound, color='red', linestyle='--', label='Lower Bound')
plt.axvline(upper_bound, color='red', linestyle='--', label='Upper Bound')
plt.legend()
plt.title('IQR Method Detection')
plt.show()
```

## Step 2: The Z-Score Method

If your feature is known to follow a Gaussian (Normal) Distribution, mapping values to Z-scores tells you how many Standard Deviations ($\\sigma$) a point is from the Mean ($\\mu$).

\\[
Z = \\frac{X - \\mu}{\\sigma}
\\]

A Z-score $> 3$ or $< -3$ is traditionally flagged.

```python
from scipy import stats

z_scores = np.abs(stats.zscore(data))
z_outliers = data[z_scores > 3]

print(f"Detected {len(z_outliers)} outliers using Z-Score.")
```

!!! warning 
    Do not use Z-Scores if your data is highly skewed (e.g., Salary data). A right-skewed tail will massively inflate the standard deviation, breaking the $Z=3$ rule of thumb. Use IQR instead.

## Step 3: Isolation Forest (Machine Learning Approach)

Unlike univariate statistical rules, `Isolation Forest` can detect multidimensional anomalies automatically by conceptually isolating instances via random feature splitting.

```python
from sklearn.ensemble import IsolationForest

# Create 2D data
X = pd.DataFrame({'Feature1': data, 'Feature2': np.random.normal(20, 5, len(data))})

iso = IsolationForest(contamination=0.01, random_state=42)
# Predictions returns 1 for inliers, -1 for outliers
predictions = iso.fit_predict(X)

X['Anomaly'] = predictions

plt.scatter(X['Feature1'], X['Feature2'], c=X['Anomaly'], cmap='viridis', alpha=0.7)
plt.title('Isolation Forest Outliers')
plt.show()
```

## Step 4: Treatment Strategies

Once detected, you must formulate a plan:

1. **Drop:** Only acceptable if the outlier is a genuine data entry error (e.g. Age = 999).
2. **Cap (Winsorization):** Replace extreme values with the boundary thresholds.
3. **Transform:** Apply a Log Transformation ($log(x)$) or Box-Cox transformation to squash the scale without losing the ordinal value.

```python
# Capping Example
data_capped = np.clip(data, lower_bound, upper_bound)
```

## Summary

Outlier detection requires domain knowledge. Never blindly drop data without investigating its origin—in cybersecurity or medical systems, the anomaly is precisely what you are trying to find!
